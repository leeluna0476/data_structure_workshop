# List 리스트
- **순서를 가진** 항목들의 모임.
  - a sequence of objects
- 배열 또는 연결된 구조로 구현할 수 있다.
- 큐, 스택과 다른 점은, 임의의 위치에서 삽입/삭제가 가능하다. 그저 순서를 가질 뿐이다.

## 리스트의 추상 자료형
- 데이터: **같은 유형의 요소들**의 순서 있는 모임.
- 연산:
```
insert(pos, e) : pos 위치에 새로운 요소 e를 삽입한다.
delete(pos)    : pos 위치에 있는 요소를 꺼내고(삭제) 반환한다.
is_empty()     : 리스트가 비어 있는지 검사한다.
is_full()      : 리스트가 가득 차 있는지 검사한다.
get_entry(pos) : pos 위치에 있는 요소를 반환한다.
```
### insert
- 기존 항목들이 순서대로 나열되어 pos 위치가 어디인지 알 수 있어야 한다.
- 항목들이 연속적으로 위치해야 한다.

## Array List VS Linked List

### 중간에 `insert`할 때
- 배열은 pos부터 끝까지 한칸씩 뒤로 옮기고, pos에 새 항목 추가.
- 연결 구조는 원하는 위치를 노드 하나씩 넘어가면서 찾은 다음, 기존 연결을 새 노드로 바꿔주면 된다. 연결 수정하는 순서 중요.

## 리스트의 활용
- 스택, 큐, 덱으로 활용할 수 있다.

## 추가해볼 만한 연산
- append(e)       : 맨 뒤에 항목 추가
- pop()           : 맨 뒤의 항목 꺼내어 반환
- replace(pos, e) : pos 위치의 항목을 e로 교체
